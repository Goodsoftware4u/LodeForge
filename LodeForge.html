Drive Cloning ✨
Makes a perfect byte-for-byte twin of one drive onto another — your whole OS, files, partitions, the works. Ideal for upgrading to a new SSD, cloning game installs, or rescuing data from a drive on its way out. The destination drive gets wiped clean first.

FULL DESKTOP FUNCTIONALITY

  • Source drive picker (drop-down with every physical drive: model + capacity + interface)
  • Destination drive picker — separate dropdown, defaults to the second drive
  • 🔄 Refresh drive list button — re-enumerates Win32_DiskDrive in case you plugged in a USB
  • PS4 mode checkbox — extended retries for encrypted / unusual drives (works for PS4 raw byte clones, but can't decrypt — the key is fused to the console's APU)
  • Dead-drive recovery checkbox — skip bad sectors instead of stopping the clone (settable as default in Settings → Dead Drive Recovery)
  • Final confirmation prompt — shows destination label "ERASE AND OVERWRITE: <label>" with Yes/No
  • Live progress bar with rounded corners and percentage fill
  • ▶ Start Clone / ✕ Cancel buttons — Cancel flips the cancellation token cooperatively
  • End-of-run message: "✓ Done — X.XX GB copied, N bad sectors skipped" (or "✗ <error>")
  • Uses CreateFile with FILE_FLAG_NO_BUFFERING for raw physical disk access; reads/writes 1 MB chunks
Drive Formatter 🧽
Wipes a drive clean and lays down a fresh file system. NTFS for Windows, exFAT for cross-platform (Mac / Linux / consoles), or FAT32 for old hardware.

FULL DESKTOP FUNCTIONALITY

  • Drive letter picker (every ready logical drive)
  • File system: NTFS (Windows) / exFAT (cross-platform) / FAT32 (old devices)
  • Partition style: GPT (modern, >2 TB) / MBR (legacy BIOS)
  • Volume label text field — defaults to "DATA"
  • Quick format checkbox (default on)
  • Final confirmation prompt: "ERASE <label>? As <FS> labelled '<label>'. All existing data will be lost. This cannot be undone."
  • Runs diskpart to convert partition style, then format.com with the file-system + label flags
  • Refreshes the letter list after success
Grow Your Partition 📐
Just cloned to a bigger drive but it still shows the old size? This stretches the selected partition into all the unused space sitting on the same disk. Non-destructive — your files stay exactly where they are.

FULL DESKTOP FUNCTIONALITY

  • Partition picker — every ready logical drive
  • Runs diskpart with: select volume X → extend
  • Pulls in all contiguous free space on the same physical disk
  • Same disk only — not a partition mover (use third-party tools for cross-disk moves)
  • Refreshes the letter list with the new size after success
Speed Check ⚡
Writes a test file to your drive, reads it back, reports real-world write + read MB/s. Catches USB ports running at 2.0 speeds when they should be 3.0, or flash drives whose NAND has worn out.

FULL DESKTOP FUNCTIONALITY

  • Drive picker (every ready logical drive)
  • Test size: 128 MB (quick) / 512 MB (balanced) / 1024 MB (accurate) / 2048 MB (thorough)
  • Sequential write of N MB to a temp file using buffered I/O
  • Then sequential read of the same file
  • Reports: Write speed: X MB/s, Read speed: Y MB/s, Average: Z MB/s
  • Speed-class grading: ≥400 MB/s — USB 3.2 Gen2 ≥120 MB/s — USB 3.0/3.1 ≥25 MB/s — USB 2.0 <25 MB/s — degraded / replace
  • Cleans up the temp file after the test

Make a Bootable USB 🚀

⚠ Not available in the web edition
Writing raw bytes to a physical USB stick requires kernel-level disk access. Use LodeForge.exe (or Rufus) on Windows — github.com/Goodsoftware4u/Lodeforge.
How it works
Turns any ISO file into a bootable USB stick — reinstall Linux (Ubuntu / Mint / Fedora), spin up a recovery tool, or repair a dead PC. The USB gets fully wiped first, then the ISO is written sector-by-sector.

FULL DESKTOP FUNCTIONALITY Rufus-style

  • Step 1 — Browse for ISO → opens an OpenFileDialog filtered to *.iso. Shows filename + size after select.
  • Step 2 — Target USB drive → drive picker with full model + capacity labels (will be erased)
  • 🔄 Refresh drive list → re-enumerates physical drives so a freshly-plugged USB shows up
  • Step 3 — Partition scheme → GPT (modern UEFI) or MBR (legacy BIOS)
  • Target system → UEFI (modern) / BIOS or UEFI / BIOS (legacy)
  • Volume label → text field, defaults to LODEFORGE
  • Verify after write → checkbox: after write completes, re-reads every byte back from the USB and compares to the source ISO; reports the exact byte offset if anything mismatches
  • Final confirmation prompt → "ERASE AND WRITE ISO TO: <drive label>. All existing data will be lost. This cannot be undone."
  • Split progress bar → 0–50% during write, 50–100% during verify (if enabled)
  • End-of-run message → "✓ Done — XX MB written + verified. USB is bootable."
  • Uses CreateFile on \\.\PhysicalDriveN with FILE_FLAG_NO_BUFFERING for raw sector writes; cancellable mid-flight
  • For ISOs > 4 GB (Windows installer images), recommended to also try Rufus directly — its NTFS+UEFI shim isn't replicated here

Stress Test & Benchmark 🔥

How it works
Pushes every available browser thread to 100% with a tight math loop AND a 256 MB RAM thrash (each worker holds a 32 MB Float64Array and reads/writes random offsets — same workout pattern as the desktop). 5 presets, live progress, cancel any time. Reports a benchmark score (ops/sec across all threads), a 1–10 silicon rating from per-thread throughput, and a thermal-pressure check derived from score variance over the run. Desktop adds real CPU temperature sensing via ACPI — flags throttle if the CPU sags under load or hits 85 °C+, with the silicon rating losing 1.5 points if throttling is detected.

Performance History 📊

How it works
Quietly logs your CPU, RAM, and temps every 2 seconds in the background. Great for catching sneaky stuff — temp spikes while gaming, memory leaks that creep up over hours, or just seeing how your system behaves over time. The web edition charts browser FPS + JS heap every ½ s as a stand-in (240-sample rolling window). Desktop logs the real sensors to %LOCALAPPDATA%\LodeForge\perf.csv.

FULL DESKTOP FUNCTIONALITY

  • 240-sample rolling window — 8 minutes of history at 2 s tick rate
  • 3 series — CPU% (purple), RAM% (green), Temp °C clamped 0–100 (amber)
  • Auto-refresh every 5 s — chart redraws from the latest CSV slice
  • Stats line: Samples: N · Avg CPU: X% · Max CPU: Y% · Avg RAM: Z% · Window: HH:MM–HH:MM
  • 🔄 Refresh — re-reads perf.csv on demand
  • 💾 Export CSV — opens a Save dialog and copies perf.csv wherever you want
  • 🗑 Clear Log — confirms first, then truncates perf.csv back to its header
  • The CSV gets written every 2 s by a background timer in MainForm: ts,cpu,ram,temp\n rows — runs whether the History tab is open or not

Settings ⚙

⚠ Web edition is read-only
All these toggles need OS-level access (system tray, powercfg, always-on-top windows, TCP socket clients, embedded HTTP server) that a sandboxed browser tab can't do. Settings auto-save on flip in the desktop — no Save button. Below: full breakdown of what each toggle does.

DESKTOP SETTINGS preview

🥷 STEALTH MODE — HOW IT WORKS

On flip: registers a NotifyIcon with a context menu (Open LodeForge / Exit), hides the main form (ShowInTaskbar = false, WindowState = Minimized), and unregisters from the taskbar. Double-click the tray icon to restore. Survives between launches — re-applied at startup.

🔋 LAPTOP POWER GUARD — HOW IT WORKS

Polls SystemInformation.PowerStatus every 30 seconds. When it sees PowerLineStatus.Offline AND BatteryLifePercent < 20%, it fires powercfg /setactive a1841308-3541-4fab-bc81-f71556f20b4a (the Power Saver scheme GUID). One-shot per AC-disconnect — won't keep flipping if you bounce the toggle.

🌡 PERFORMANCE OVERLAY — HOW IT WORKS

Spawns a borderless 240×78 always-on-top window at (20, 20) with rounded corners, 92% opacity, and Cascadia Mono text. Updates every 1.5 s with CPU XX% TEMP XX°C / GPU XX% RAM XX%. Black background, 2 px purple outline, white text for high contrast against any wallpaper or game. Closes cleanly on flip-off — no stale window.

💀 DEAD DRIVE RECOVERY — HOW IT WORKS

Sets the "Dead-drive recovery — skip bad sectors instead of stopping" checkbox in the Storage Suite → Clone tab to be ON by default for new sessions. Existing Clone tab needs reopening to pick up. No actual behavior change to clones already in flight.

🌈 RGB STATUS SYNC — HOW IT WORKS

Implements the OpenRGB SDK as a TCP client. Connects to 127.0.0.1:6742 (silent if OpenRGB isn't running), sends a NET_SET_CLIENT_NAME = "LodeForge" packet, queries NET_REQUEST_CONTROLLER_COUNT, then ticks every 3 s: reads CPU temp, picks a color (cyan ≤ 0 unknown, green ≤ 60 °C, amber 60–80, red > 80), and broadcasts NET_RGBCONTROLLER_UPDATESINGLELED to every controller's first LED.

📱 PHONE DASHBOARD — HOW IT WORKS

When the toggle's on, LodeForge starts an embedded HttpListener on port 8765, runs netsh advfirewall firewall add rule to open the port (one-time, on first start), and serves a live dashboard at http://<your-PC-LAN-IP>:8765/. Any device on the same Wi-Fi can visit that URL — no app install.
Endpoints: / (HTML dashboard) · /api/live (JSON: cpu_pct, ram_pct, gpu_pct, cpu_temp, ram_total_mb, ram_avail_mb, polled every 1.5 s) · /api/system (JSON: host, cpu_name, cpu_cores, cpu_threads, gpu_name, board)
